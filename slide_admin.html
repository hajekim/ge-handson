기업 내 안전한 생성형 AI 도입을 위한 데이터 격리, 권한 제어, 보안 관제 및 투자 효과(ROI) 검토 방안을 다룹니다.

⬇️ 방향키(↓) 또는 마우스 스크롤로 다음 슬라이드로 이동

GCP 프로젝트 설정 및 인프라 거버넌스

Gemini Enterprise의 안전한 전사 배포를 위해 클라우드 인프라의 환경 격리 및 권한 분리 기준을 수립합니다.

🏛️ 환경별 프로젝트 분리

단일 프로젝트 내 집중을 피하고 운영(Prod) / 개발(Dev) / 감사(Audit) 프로젝트로 격리하여 리소스 간섭 및 단일 실패 지점을 방지합니다.

🔐 최소 권한 원칙 (PoLP) 적용

일반 사용자의 서비스 사용 권한과 인프라 관리자의 설정 권한을 분리하며, Cloud Identity 조직 단위(OU) 기반으로 그룹 권한을 매핑합니다.

다중 프로젝트 가용성 아키텍처

서비스 연속성 유지 및 데이터 관리 요건 충족을 위해 멀티 리전 배포 및 데이터 암호화 구조를 적용합니다.

🌐 멀티 리전 배포 구조

  • asia-northeast3 (서울): 주요 운영 트래픽 처리 및 색인 데이터 보관
  • us-central1 (아이오와): 재해 복구(DR) 대비 및 백업 리전 활용

🛡️ 네트워크 및 암호화 통제

VPC Service Controls: API 호출 및 데이터 색인의 인터넷 통신 제어

  • Cloud KMS (CMEK): 고객 관리 암호화 키를 활용한 데이터 저장 암호화

    • 심화: VPC SC 및 KMS CMEK 관리

    사내 데이터 보호를 위한 보안 경계(Service Perimeter) 설정 및 고객 관리 암호화 키(CMEK) 운영 주기 관리입니다.

    💡 보안 경계 통제 기준

    VPC SC 경계 내부 리소스 간의 통신만 허용되며, 승인되지 않은 외부 IP 주소 및 미인증 단말에서의 데이터 입출력을 제어합니다.

    • CMEK 주기적 회전: 설정된 주기(예: 90일)에 따라 암호화 키를 자동 회전하여 암호키 노출 에스코프를 최소화합니다.

    사용자 인증 관리: WIF & Cloud Identity

    장기 보안 자격증명(`.json` 키 파일) 사용의 위험을 줄이기 위해 Workload Identity Federation (WIF) 기반 인증을 구성합니다.

    🔑 WIF 기반 키리스(Keyless) 연동

    사내 IDP(Okta, Entra ID 등)와 구글 클라우드를 OIDC 표준 프로토콜로 연결하여 정적 키 발급 없이 단기 일시 토큰으로 인증을 수행합니다.

    👥 인사 조직 정보 동기화

    인사 관리 시스템(HRIS)과의 동기화를 통해 퇴직 및 부서 이동 시 IAM 접근 권한이 자동 갱신 및 해지되는 파이프라인을 운영합니다.

    데이터 커넥터 연동: M365 & ServiceNow

    Google Workspace 외에 Microsoft 365(Outlook, OneDrive) 및 ServiceNow 사내 시스템 데이터의 색인 및 검색을 연계합니다.

    🔌 M365 엔터프라이즈 커넥터

    임직원의 이메일 내역 및 원드라이브 첨부 문서를 색인하여 업무 콘솔 내에서 통합 RAG 검색 환경을 지원합니다.

    🛡️ 접근 제어 목록(ACL) 동기화

    원본 데이터 소스의 개별 사용자 접근 권한(ACL) 정보가 색인에 동기화되므로, 원본 시스템에서 열람 권한이 없는 데이터는 검색 결과에 표출되지 않습니다.

    심화: Actions (쓰기 권한) 거버넌스

    에이전트가 외부 시스템에 데이터를 입력하거나 수정하는 '쓰기 Action' 수행 시의 보안 통제 매커니즘을 설명합니다.

    ⚠️ 승인 단계 필수화 (Human-in-the-Loop)

    외부 시스템에 변경을 가하는 Action 트리거 발생 시, 자동 처리를 일시 중지하고 사용자 인터페이스에 [실행 승인 요청] 창을 표출합니다.

    • 권한 토큰 분리: 조회(Read-only) 권한과 변경(Write Action) 권한을 IAM 정책에서 분리 적용합니다.
    • 감사 내역 기록: Action 실행자, 승인 일시, 파라미터 정보가 Cloud Audit Logs에 보관됩니다.

    전사 에이전트 레지스트리 (Agent Registry)

    사내 각 조직에서 배포한 커스텀 에이전트의 현황을 중앙 관리 콘솔에서 모니터링하고 권한을 통제합니다.

    👁️ 활성 상태 모니터링

    등록된 에이전트의 배포 수, 부서별 활성 사용량, 매핑된 데이터 소스 연결 현황을 중앙에서 조회합니다.

    🔐 ACL 접근 범위 통제

    인사, 재무 등 민감 데이터를 다루는 에이전트는 인가된 특정 보안 그룹만 접근할 수 있도록 접근 제어 목록을 설정합니다.

    🛑 에이전트 비활성화 (Kill Switch)

    보안 정책 위반이나 비정상 호출이 감지된 에이전트를 콘솔에서 즉시 '비활성화(Disable)'하여 호출을 차단합니다.

    실습: 에이전트 ACL 감사 및 비활성화

    관리자 콘솔에서 정책 위반 에이전트를 확인하고 활성 상태를 제어하는 실습 절차입니다.

    📋 통제 실습 단계

    • 1단계 (조회): Agent Registry 목록에서 대상 에이전트(`[비공식_재무_급여_계산_봇]`) 선택
    • 2단계 (권한 검토): 접근 권한 그룹이 `All Employees(전사)`로 과다 설정되어 있는지 감사
    • 3단계 (비활성화): 상단의 [Disable Agent] 버튼을 클릭하여 서비스 접근을 차단하고 소유자에게 알림 발송

    Model Armor: 보안 방화벽 3대 지표

    실시간 데이터 입출력 방화벽인 Model Armor를 통해 제어하는 주요 데이터 검열 지표입니다.

    🛡️ 1. 민감 정보(PII) 탐지

    주민등록번호, 전화번호, 금융 정보 등 개인식별정보(PII) 패턴 감지 시 마스킹 또는 요청 차단 규칙을 적용합니다.

    🚫 2. 프롬프트 인젝션 방어

    시스템 지시어 우회(Jailbreak) 및 시스템 프롬프트 유출 시도를 탐지하여 모델 인퍼런스 전송 전 필터링합니다.

    ☣️ 3. 유해 콘텐츠 검열

    욕설, 혐오 표현, 비윤리적 지시문 등 기업 업무에 부적합한 입력 및 출력 데이터를 검출합니다.

    실습: Model Armor 임계값 정책 설정

    조직 단위(OU) 및 부서 특성에 맞춰 Model Armor 필터링 강도를 조정하는 설정 방안입니다.

    💡 조직별 차등 제어 정책

    • 고객 상담 부서 (CS): PII 필터 강도를 `High(엄격)`로 설정하여 대화 내 고객 식별정보 노출 방지
    • 연구 개발 부서 (R&D): 소스 코드 및 기술 시나리오 문맥 오탐을 줄이기 위해 구문 필터 임계값을 조정하여 개발 효율성 유지

    감사 로그 SIEM 연계: BigQuery 관제

    에이전트 호출, 문서 색인 참조 내역은 Cloud Audit Logs를 거쳐 BigQuery 데이터셋으로 전송됩니다.

    🔍 호출 내역 추적성

    호출 일시, 요청 사용자 이메일, 사용된 에이전트 ID, 참조된 문서 URI 등 상세 감사 정보를 보관 및 조회합니다.

    🚨 SIEM 툴 연동 분석

    Splunk, Chronicle 등 사내 SIEM 시스템과 쿼리를 연동하여 단기간 내 다량 문서 참조 발생 시 보안 이벤트 알림을 생성합니다.

    실습: BigQuery 감사 로그 조회 SQL

    관리자가 BigQuery 콘솔에서 단시간 내 다수 문서를 참조한 호출 내역을 조회하는 쿼리 실습입니다.

    📋 SIEM 탐지 SQL 예제

    SELECT prolog.timestamp, prolog.caller_ip, prolog.user_email, prolog.agent_id, prolog.action_type, prolog.query_text, prolog.cited_docs FROM `sys-audit-prod-01.gemini_siem.cloudaudit_googleapis_com_data_access` AS prolog WHERE prolog.timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 1 HOUR) AND prolog.action_type = 'RAG_CITATION_QUERY' AND ARRAY_LENGTH(prolog.cited_docs) >= 15 ORDER BY prolog.timestamp DESC;

    💡 결과: 설정된 시간 범위 내 기준치(15개) 이상의 문서를 인용한 사용자 IP 및 요청 이력 리스트가 출력됩니다.

    재무 검토: ROI 시뮬레이션 모델

    AI 솔루션 도입 검토 시 경영진 보고를 위해 활용하는 정량적 비용 및 효용(ROI) 산출 기준입니다.

    💰 비용 및 절감 파라미터

    • 예상 지출: 구독 라이선스 비용 + 클라우드 인프라 운용 비용
    • 업무 절감 가정: 인당 월간 반복 업무 절감 시간(Hour)
    • 환산 가치: 평균 임금 환산액을 적용한 연간 절감 기대 효과

    📈 손익분기점 (BEP) 시뮬레이션

    연도별 예상 서비스 활용률(1차년도 온보딩 ~ 3차년도 정착)을 반영하여 누적 순현재가치(NPV) 및 BEP 시점을 산출합니다.

    NotebookLM Enterprise 거버넌스 설정

    기업용 NotebookLM의 팀 도서관 데이터 외부 공유 통제 및 API 연동 기준을 설명합니다.

    🏛️ 엔터프라이즈 도서관 권한 제어

    • 공유 도메인 제한: 사내 인가 도메인(`.company.com`) 이외의 외부 계정으로 도서관 세션 공유 차단
    • 내보내기 통제: 업로드된 원본 문서 소스의 외부 다운로드 및 복사 제한 기능

    🌐 사내 시스템 REST API 연계

    그룹웨어 및 사내 업무 포털에서 REST API를 호출하여 NotebookLM 도서관에 정형/비정형 문서를 자동 업로드하는 표준을 적용합니다.
